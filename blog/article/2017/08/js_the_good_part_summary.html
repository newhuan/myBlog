《Javascript语言精粹》——总结

20150818

parseInt(num, base);//最好总是提供base参数用于指定转换的number的进制，避免由0或0x开头的字符串被转化为8或16进制数字。

函数在任何位置（包括局部作用域内）进行函数调用的时候(foo();)，其中的this指针都指向全局window对象； 但是，如果某个函数在函数体内应用了严格模式，那么此函数自身以及其内部定义的函数进行'函数调用'(foo();)时，this指针指向undefined或null（根据js引擎解析的不同而不同）

尽量使用双引号而不是单引号 

                        
Function.prototype.method = function(name, func){
//再此function对象的prototype上定义一个名字是name，值是func的属性；
    this.prototype[name] = func;
    return this;//级联
}

字面量（literal）是用于表达源代码中一个固定值的表示法（notation）

尝试检索一个undefined值上的属性会报一个TypeError 

                        
Object.beget = function (o) {
//创建一个以o对象作为原型链的新对象；Object.create();
    var F = function () {};
    F.prototype = o;
    return new F();
}

编程 => 将一组需求分解成一组函数与数据结构（函数式编程？）

表现力：代码能很清晰的表达自己做的事

自定义事件：键名为事件名，值为一个数组，元素是事件处理方法；添加时，如果没有此键，初始化并赋值为数组；再次添加时，向数组中push；触发时，检索事件名，按数组顺序依次调用数组中的事件处理方法；

当属性名是小而连续的整数时，使用数组，否则使用对象；数组的splice方法比较慢；js数组的内存地址不是连续的，本质是一个键名为连续整数的对象；

return语句后面的表达式的开始部分必须和return处于同一行

使用关键字作为key时使用下标表示法(['case'])(但是还是尽可能的不要使用关键字作为key)

为了避免浮点数异常，需要处理高精度小数时，可以转化为整数进行整数运算，再把结果转为小数(*100/100)

尽量避免continue关键字，影响性能
